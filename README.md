# 🩺 Тыжархитектор

> Карточный тренажёр по программной архитектуре. Подзаголовок — *«Диагноз и лекарство»*.

Онлайн-версия настольной игры по программной архитектуре.
Тянешь карту-**симптом** (проблема системы с цифрами) → подбираешь карту-**шаблон**
(архитектурное решение с ценой) → приложение открывает ключ и считает очки.
Половина соблазнительных лекарств лечит не то, а часть болезней лечится тем, что
**НЕ усложняешь** (оставить монолитом, взять готовое).

> Методичка-первоисточник (правила, колоды, ключ ответов) живёт отдельно, в
> Obsidian-хранилище: `architecture_games/Диагноз и лекарство/03_диагноз_и_лекарство.md`.
> Здесь — только приложение. Карты в `src/cards.js` — зеркало методички.

## Режимы

| Режим | Кто «ведущий» | Скрытые руки | Что нужно |
|---|---|---|---|
| 🧍 **Соло (пасьянс)** | приложение (ключ = данные) | — | браузер |
| 👥 **Хотсит / проектор** | человек за общим экраном | нет | браузер + общий экран/созвон |
| 🌐 **Мультиплеер** *(Этап 2, в планах)* | хост или приложение | да | + Node-сервер (WebSocket) |

## Как запустить

### 1. Локально, двойным кликом
Открой `index.html` в браузере. Всё. Сборки и сервера не нужно — карты подключены
как JS-модуль (`src/cards.js`), а не через `fetch`, поэтому `file://` не упирается в CORS.

### 2. Локально через статик-сервер
Если браузер строг к ES-модулям по `file://`:
```bash
python -m http.server 8000
# открой http://localhost:8000
```
(или `npx serve`, если есть Node).

### 3. GitHub Pages (бесплатный онлайн, Этап 1)
Игра статична, поэтому Pages её держит без всякой сборки:
1. запушь репозиторий на GitHub;
2. **Settings → Pages → Build and deployment → Branch: `main`, папка `/ (root)`**;
3. играй по выданному `https://<user>.github.io/<repo>/` и шарь ссылку на воркшоп.

## Тесты
Без зависимостей — встроенный тест-раннер Node:
```bash
node --test
# или
npm test
```
Покрыто игровое ядро: подсчёт очков, штраф за оверинжиниринг на ловушках, резолв
ключа, раздача без повторов, контракт транспорта.

## Архитектура — «ядро + транспорты» (ports & adapters)

Тот самый паттерн, что лежит в колоде самой игры: чистое **ядро** в центре,
сменный **адаптер транспорта** снаружи.

```
            ┌─────────────────────────────┐
            │   game-core.js (ЯДРО)        │  правила, колоды, ключ, счёт
            │   чистая логика, без DOM/сети │  ← тестируется напрямую
            └──────────────┬──────────────┘
                           │  reduce(state, move)
        ┌──────── transport.js (ПОРТ) ────────┐
        ▼                                      ▼
 transport-local.js                  NetworkTransport (заглушка)
 состояние в одной вкладке            WebSocket → сервер (Этап 2)
        │                                      │
        ▼                                      ▼
   соло + хотсит                       мультиплеер, скрытые руки
                 ▲
                 │  getState / applyMove / subscribe
            ui.js (рендер, не знает про сеть)
```

| Файл | Роль |
|---|---|
| `src/cards.js` | данные карт (симптомы, шаблоны, ключ ответов, комбо) — зеркало методички |
| `src/game-core.js` | игровое ядро: `createGame`, `reduce`, `scorePlay`, `answerKey` — чистое, без DOM/сети |
| `src/transport.js` | интерфейс транспорта (порт) + заглушка `NetworkTransport` для Этапа 2 |
| `src/transport-local.js` | локальный адаптер: соло и хотсит без сервера |
| `src/ui.js` | рендер и взаимодействие; работает только через интерфейс транспорта |
| `src/main.js` | точка входа: собирает Local-транспорт + UI |
| `test/game-core.test.js` | тесты ядра (`node --test`) |

Почему так: UI и ядро написаны против **интерфейса транспорта**
(`getState` / `applyMove` / `subscribe`). Этап 1 подключает `LocalTransport`.
Этап 2 подключит `NetworkTransport` с тем же интерфейсом — **ни ядро, ни UI не
переписываются**.

## Правила подсчёта (кратко)

- Точное лекарство по ключу — **+3**; допустимая альтернатива — **+2**.
- Полное комбо там, где нужна связка, — **+5** (лучшее +3 и бонус +2); половина связки — **+2**.
- Карта сдержанности на 🪤-ловушке — **+3**; тяжёлый (распределённый/дорогой) шаблон на ловушке — **−2**.
- «Лечит симптом, а не причину» — **0**.

Тяжёлые шаблоны помечены значком ⚖ — на ловушках это и есть мишень игры (оверинжиниринг).

## Roadmap — Этап 2 (мультиплеер)

Отдельной задачей, ядро не трогаем:
- маленький **Node + `ws`** сервер: комнаты, коды подключения, раздача **скрытых
  рук**, рассылка ходов; `game-core.reduce` переезжает на сервер как есть;
- реализовать `NetworkTransport` (см. заглушку в `src/transport.js`) поверх WebSocket;
- GitHub Pages держит только статику, поэтому сервер деплоится отдельно
  (Render / Railway / Fly), а его URL прокидывается в `NetworkTransport`; клиент
  остаётся на Pages.
